# sorter64

Streaming PGN Chunker + Sorter + Filter for the e-schachdojo pipeline.

## Installation

```bash
cd sorter64
npm install
```

No heavy dependencies; tests use Node's built-in test runner.

## Usage

Start the GUI launcher (file/folder picker):

```bash
.\dist\run.bat
```

Run via CLI using the launcher:

```bash
.\dist\run.bat --input lichess.pgn --games-per-chunk 5000
```

Run via CLI directly (Node entrypoint):

```bash
node .\bin\sorter64.js --input lichess.pgn --games-per-chunk 5000
```

With Elo bins and time control filtering:

```bash
node .\bin\sorter64.js --input lichess.pgn --elo-bins "800-1200,1200-1600,1600-2000,2000-9999" --time-controls blitz,rapid
```

Date and rated filters + dedupe:

```bash
node .\bin\sorter64.js --input lichess.pgn --dedupe --rated only --since 2021-01-01 --until 2021-12-31
```

## Filters

- Time controls: based on `TimeControl` header.
- bullet: base < 180s
- blitz: 180s�479s
- rapid: 480s�1499s
- classical: >= 1500s
- unknown: only kept if explicitly included
- Rated: heuristic on `Event`, `Site`, and `Rated` headers; looks for �Rated� or �Casual�.
- Moves: `--min-moves`/`--max-moves` use a stable SAN token count.
- Dates: `UTCDate` preferred, falls back to `Date`.

## Outputs

- Chunks: `out/chunks/<prefix>_0001.pgn`
- Elo bins: `out/bins/<bin>/<prefix>_0001.pgn`
- Skipped log: `out/skipped.txt`
- Summary: `out/report.json`

## Pipeline Context

This tool sits in the pipeline:

Extractor64 -> Converter -> Backend

`sorter64` produces clean, fully-delimited PGN chunks to ensure downstream tools remain stable.

## Launchers

`dist\run.bat` starts the GUI when called with no arguments and starts the CLI when called with arguments. The launchers in `dist\` are generated by the build, so do not edit them manually.
